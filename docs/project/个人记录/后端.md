# 后端

这份文档用于记录我对 SysY 编译器 RISC-V 后端部分的个人理解。后续可以围绕栈帧布局、调用约定、指令选择、全局数据输出、数组寻址、大栈偏移和寄存器分配演进方向继续补充。

## 1. 记录方向

建议后续从这些问题展开：

- RISC-V 后端为什么当前采用栈式模型？
- `FunctionFrame` 如何预扫描函数并分配对象槽、值槽、返回地址和 outgoing argument area？
- 函数调用时，参数如何分配到 `a0-a7` 和栈上传参区域？
- 为什么包含 call 的函数需要保存 `ra`？
- `materialize_value` 和 `materialize_pointer` 分别解决什么问题？
- `getptr` / `getelemptr` 如何下降到 RISC-V 地址计算？
- 大 offset 超过 imm12 范围时如何处理？
- 后续如何从栈式后端演进到虚拟寄存器、活跃区间和 linear scan 寄存器分配？

## 2. 面试讲述入口

可以先用一句话概括：

> 当前 RISC-V 后端优先保证正确性，采用栈式代码生成策略：先根据 IRFunction 预扫描并计算栈帧布局，再逐条把 IR 指令翻译成 RISC-V 汇编；函数调用遵守 `a0-a7` 参数寄存器和栈上传参规则，必要时保存 `ra`。
